# Rust 基础语法示例

这段代码演示了 Rust 在类型转换、字面量、字符处理和数组等方面与 C/C++ 的主要区别。

## 运行方式

```bash
rustc main.rs
./main
```

或使用 `cargo`（需要先 `cargo init` 并将代码放入 `src/main.rs`）：

```bash
cargo run
```

## 代码要点说明

### 1. 类型转换与字面量后缀

- **显式转换**：Rust 不会像 C++ 那样隐式转换类型，必须使用 `as` 关键字。
- **字面量后缀**：可以直接在数字后加 `_type` 表明类型，例如 `100_u8`、`0.12_f32`。仅限基本数据类型。
- **数字分隔符**：可以用下划线提升可读性，如 `100_000_000`、`2.23_34_f64`。
- **ASCII 字节**：存储 ASCII 字符用 `b'X'` 形式，得到的是 `u8`。不能写成 `'+'_u8`。

```rust
let a = b'a';
let aa = b'A';
let x = 43_u8;
let y = 43;
```

### 2. 字符 `char` 与 ASCII 转换

- `u8` 可以通过 `as` 转成 `char`（例如 `x as char`），也可以链式转换：`y as u8 as char`。
- **重要**：Rust 的 `char` 占 4 个字节，代表一个 Unicode 标量值，而 C/C++ 中通常占 1 个字节。

### 3. Unicode 字符

- 使用 `\u{...}` 转义，**必须加花括号**，例如 `'\u{4f60}'` 表示“你”。

```rust
let chinese = '\u{4f60}'; // 你
println!("{}", chinese);
```

### 4. 安全的数字到字符转换

- 不是所有 `u32` 值都有对应字符，所以 `u32 as char` 会编译错误。
- 正确做法是使用 `char::from_u32()`，它返回 `Option<char>`，表示可能转换失败。

```rust
let u32v = 0xffff_u32;
let ch = char::from_u32(u32v);  // 返回 Some 或 None
println!("ch:{:?}", ch);
```

### 5. 数组

- 声明：`let arr : [Type; size] = [v1, v2, ...];` ，类型和长度必须同时指定。
- 大小在**编译期**确定，不支持 VLA（可变长数组）。
- 不可变：默认不能修改元素，需要修改时加 `mut`。
- 初始化：
  - 全部相同值：`[0; 10]` 产生长度为 10 且全为 0 的数组。
  - 指定类型：`[10_u8; 20]`。
- 索引从 0 开始，访问 `arr[2]`。

```rust
let ints = [1, 2, 3, 4];
let arr = [0; 10];          // 10 个 0
let arr = [10_u8; 20];     // 20 个 10u8
```

### 6. 格式化打印

- 打印数组不能直接用 `{}`，需要调试格式 `{:?}` 或美化调试 `{:#?}`。
- 打印 `Option` 等复合类型也用 `{:?}`。

```rust
println!("{:?}", ints);     // [1, 2, 3, 4]
println!("{:#?}", ints);    // 多行美化输出
```

### 7. 运算符

- `+` `-` `*` `/` `%` 语法和 C++ 一样，但**类型必须匹配**才能运算，不会自动提升或截断。
